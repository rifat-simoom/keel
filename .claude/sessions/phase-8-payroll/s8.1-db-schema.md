## Your task
Create the Alembic migration and SQLAlchemy models for `employees` and `payroll_runs` tables as specified in SPEC.md Phase 8. Add `deleted_at`, `created_by`, `updated_by` to both. Add `payroll_line_items` table: `id UUID PK`, `payroll_run_id UUID FK`, `employee_id UUID FK`, `gross_pay NUMERIC(12,2)`, `paye_tax NUMERIC(12,2)`, `employee_nic NUMERIC(12,2)`, `employer_nic NUMERIC(12,2)`, `net_pay NUMERIC(12,2)`, `tax_code TEXT DEFAULT '1257L'`, `ni_category TEXT DEFAULT 'A'`. Add `payslips` table: `id UUID PK`, `payroll_line_item_id UUID FK`, `pdf_key TEXT` (MinIO key), `issued_at TIMESTAMPTZ`. Add indexes on `employees.company_id`, `payroll_runs.company_id`, `payroll_runs.pay_period`, `payroll_runs.status`.

## Constraints
- All monetary columns: `NUMERIC(12,2)` — never float (FP-2)
- `payroll_runs.status` CHECK constraint: `draft`, `approved`, `submitted` only
- `employees.ni_number` stored encrypted (use `pgcrypto` `pgp_sym_encrypt` or note it as a TODO for security review if not available)
- `employees.deleted_at` present — payroll records must never be hard-deleted (FR-22: 3-year retention)
- `payroll_runs.submitted_to_hmrc` defaults to FALSE — set to TRUE only after RTI submission (FR-19)
- Migration reversible

## Done when
- `alembic upgrade head` creates all four tables cleanly
- `alembic downgrade -1` reverses cleanly
- `pytest backend/services/payroll/tests/test_models.py -v` passes (all monetary fields are Decimal, status constraint, soft delete)
- `make verify` runs alembic + pytest and exits 0

## Do NOT touch
- PAYE/NIC calculator (s8.2)
- RTI submission (s8.3)
- API endpoints or UI
