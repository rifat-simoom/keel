# Session: Payroll — Web UI
## Phase
8 — Payroll (Post-MVP)

## Context files to load

## Your task
Build the complete payroll web UI: (1) `web/src/pages/PayrollPage.tsx` — two-tab layout: Employees and Pay Runs. (2) Employees tab: table with columns (name, NI number masked `****1234`, tax code, salary, start date), "Add Employee" button opens a slide-out form, "Edit" soft-edits details, "Remove" shows a confirmation and soft-deletes. (3) Pay Runs tab: list of past runs with status badges (draft/approved/submitted), "Run Payroll" button creates a new draft, draft run shows per-employee breakdown table (gross, PAYE, employee NIC, employer NIC, net), "Approve" button (owner only) with confirmation, "Submit to HMRC" button (approved runs only) with a warning: "This will submit to HMRC and cannot be undone". (4) Submitted runs show lock icon, are read-only, and display the RTI correlation ID. (5) `web/src/pages/PayrollPage.tsx` accessible from sidebar "Payroll" link (already exists as placeholder).

## Constraints
- NI numbers displayed masked in all list/table views — full number shown only in edit form (PII)
- "Submit to HMRC" button: requires two-step confirmation — one click shows modal, second click submits (FR-19 gravity)
- Employer NIC column labelled "Company NIC cost" — clearly distinguished from employee NIC
- Submitted run: all fields disabled, RTI correlation ID shown prominently
- `owner` role only sees Approve and Submit buttons — other roles see read-only
- All monetary values via `@keel/utils` formatCurrency

## Done when
- Employee CRUD works end-to-end
- Draft payroll run shows correct PAYE/NIC calculations per employee
- Approve and Submit buttons follow the correct role-gated flow
- Submitted run shows lock + RTI correlation ID
- `pnpm --filter web test --run` passes payroll table, employee form, and run status tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Mobile payroll UI (out of MVP scope)
- PAYE calculator or RTI backend
