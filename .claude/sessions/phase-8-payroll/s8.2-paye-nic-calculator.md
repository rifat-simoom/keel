## Your task
Implement `backend/services/payroll/calculator.py` — `PAYECalculator` class with: (1) `calculate_employee(gross_salary: Decimal, tax_code: str, pay_period: str) -> EmployeePayCalculation` — returns `income_tax`, `employee_nic`, `net_pay`, `taxable_pay`. (2) `calculate_employer(gross_salary: Decimal) -> Decimal` — returns employer NIC (13.8% above £9,100 secondary threshold). (3) `calculate_payroll_run(employees: list[Employee]) -> PayrollRunCalculation` — returns `total_gross`, `total_paye`, `total_employee_nic`, `total_employer_nic`, `total_net`. Tax code `1257L` means personal allowance of £12,570. Support annualised and monthly calculation (divide annual thresholds by 12 for monthly payroll). Income tax bands applied in sequence — basic then higher then additional.

## Constraints
- All arithmetic: `Decimal` only (FP-2)
- All thresholds/rates from `shared.constants.tax` (FP-9): must add `PAYE_BASIC_RATE`, `PAYE_HIGHER_RATE`, `PAYE_ADDITIONAL_RATE`, `NIC_SECONDARY_THRESHOLD`, `NIC_EMPLOYER_RATE`, `HIGHER_RATE_THRESHOLD`, `ADDITIONAL_RATE_THRESHOLD`
- Tax code `1257L` → personal allowance `Decimal("12570")` — tax code parsing for common codes only
- Monthly calculation: annual thresholds ÷ 12, rounded to nearest penny
- `net_pay = gross_salary - income_tax - employee_nic` — verify algebraically in tests
- Employer NIC is a company cost — never deducted from `net_pay`

## Done when
- Gross £12,570/yr → zero income tax, zero employee NIC
- Gross £30,000/yr → correct income tax and NIC at basic rate
- Gross £60,000/yr → higher rate band applies correctly above £37,700 threshold
- Employer NIC correctly = 13.8% of (gross - £9,100)
- Monthly vs annual calculation gives the same annual totals when multiplied by 12
- `pytest backend/services/payroll/tests/test_calculator.py -v` covers all three income bands and monthly calc
- `make verify` runs pytest and exits 0

## Do NOT touch
- RTI submission (s8.3)
- API endpoints (s8.4)
- Web UI
