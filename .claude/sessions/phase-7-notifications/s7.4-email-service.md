# Session: Notifications — Email Service (Resend)
## Phase
7 — Notifications & Deadlines

## Context files to load
- `SPEC.md` (section: Phase 7 — Notification Types table)
- `CLAUDE.md`
- `.claude/guardrails/forbidden-patterns.md` (FP-18)
- `backend/services/notifications/models.py`

## Your task
Implement email dispatch in `backend/services/notifications/email.py` using the Resend Python SDK (`resend` package): (1) `EmailDispatcher` class with method `async send(notification: Notification, user_email: str) -> None`. (2) Create HTML email templates (as Python string templates or Jinja2) for each notification type that requires email (per SPEC table): `invoice_viewed`, `invoice_overdue`, `payment_received`, `tax_deadline_30d`, `tax_deadline_7d`, `tax_deadline_1d`, `payroll_due`. Each template must include: the notification body, a clearly labelled call-to-action button linking to `https://keelapp.co.uk{route}`, the Keel logo placeholder, and an unsubscribe footer. (3) Tax deadline emails (30d, 7d, 1d) must state the specific deadline date and amount owed (passed via `notification.metadata`).

## Constraints
- Resend API key from `RESEND_API_KEY` env var — never hardcoded (CLAUDE.md rule 14)
- From address: `Keel <notifications@keelapp.co.uk>`
- Never log email body content — log `notification.type` and `user_id` only (FP-18)
- HTML templates must be valid HTML5 with inline CSS (email clients don't support external CSS)
- Tax deadline emails: urgency reflected in subject line — "Action required: VAT return due tomorrow"
- `send()` must not raise on Resend API failure — log error, mark notification as failed in metadata
- Templates stored in `backend/services/notifications/templates/` directory

## Done when
- `EmailDispatcher.send()` calls Resend API with correct payload
- Tax deadline email subject line changes based on urgency (30d / 7d / 1d)
- Template renders correct CTA button URL including the route
- `pytest backend/services/notifications/tests/test_email.py -v` passes (mock Resend API)
- `make verify` runs pytest and exits 0

## Do NOT touch
- Push notification service (s7.3)
- Invoice PDF email attachment (already in Phase 4 — that uses Resend directly)
- In-app notifications
