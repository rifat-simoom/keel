# Session: Notifications — Mobile Push Setup
## Phase
7 — Notifications & Deadlines

- `SPEC.md` (section: Phase 7 — Mobile Push Setup, Notification Types table)

## Your task
Implement mobile push notification setup in `mobile/lib/notifications.ts` and wire it into the app: (1) `registerForPushNotificationsAsync()` — requests permission via `expo-notifications`, gets the Expo push token, calls `POST /api/v1/notifications/register-device` with the token and platform. (2) `setupNotificationHandlers()` — `addNotificationResponseReceivedListener`: when user taps a notification, reads `response.notification.request.content.data.route` and calls `router.push(route)` for deep linking. (3) Call `registerForPushNotificationsAsync()` in `mobile/app/(app)/_layout.tsx` on first authenticated load (after login confirmed). (4) `mobile/app/(app)/more/notifications.tsx` — in-app notification list screen: FlatList of recent notifications with swipe-to-delete (marks read), unread highlighted, tapping navigates to route. Pull-to-refresh. (5) Add a red badge on the "More" tab when unread count > 0.

## Constraints
- Push permission request must show a user-facing explanation before the system prompt — `expo-notifications` allows this
- If permission denied: silently skip registration (no crash), show a settings hint in the notifications screen
- Token registration called only once per app launch — check if already registered with same token (idempotent)
- Deep link navigation uses Expo Router's `router.push()` — not `Linking.openURL()`
- Unread badge on More tab uses `tabBarBadge` prop from Expo Router Tabs
- Never log the raw push token to console (FP-18)

## Done when
- App requests push permission on first load
- Tapping a received push notification deep-links to the correct screen
- In-app notification list shows recent notifications with unread highlighting
- Swipe-to-dismiss marks notification as read via API
- More tab shows red badge when unread count > 0
- `pnpm --filter mobile test --run` passes notification registration and handler tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web notification bell (s7.6)
- Push dispatch backend (s7.3)
- Other mobile screens
