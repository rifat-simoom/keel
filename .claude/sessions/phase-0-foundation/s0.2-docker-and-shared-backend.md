# Session: Docker Compose & Shared Backend Patterns
## Phase
0 — Foundation

## Context files to load
- `SPEC.md` (sections: 0.2 Docker Compose, 0.3 Shared Backend Patterns, Tech Stack — Backend)
- `CLAUDE.md`

## Your task
Create `infrastructure/docker-compose.yml` with all six services (postgres:16, rabbitmq:3-management, redis:7, keycloak:24.0, minio) with correct ports, environment variables, and volume mounts for persistence. Then create `backend/shared/` with: (1) `patterns/result.py` — the `Result[T]` dataclass, (2) `patterns/outbox.py` — the outbox publisher that writes to `outbox_events` table then publishes to RabbitMQ, (3) `middleware/auth.py` — `require_auth` and `require_role` FastAPI dependencies (JWT decode only — JWKS fetch is Phase 1), (4) `constants/tax.py` — all HMRC tax constants as named `Decimal` values, (5) `models/audit.py` — the `AuditLog` SQLAlchemy model and `write_audit_log()` helper.

## Constraints
- Keycloak image: `quay.io/keycloak/keycloak:24.0`
- MinIO command: `server /data --console-address ":9001"`
- All monetary constants in `tax.py` must be `Decimal`, never `float` (FP-9)
- `write_audit_log()` must be called inside the same DB transaction as the mutation (atomicity)
- Outbox pattern: write to DB first, background worker publishes — guarantees at-least-once delivery
- `Result[T]` must be generic with `value: Optional[T]` and `error: Optional[str]`

## Done when
- `docker compose -f infrastructure/docker-compose.yml up -d` starts all six services healthy
- `python -c "from backend.shared.patterns.result import Result; r = Result(value=1); assert r.is_success"` passes
- `python -c "from backend.shared.constants.tax import VAT_STANDARD_RATE; from decimal import Decimal; assert VAT_STANDARD_RATE == Decimal('0.20')"` passes
- `make verify` runs `docker compose config --quiet` (validates YAML) and `pytest backend/shared/tests/ -v` and exits 0

## Do NOT touch
- `packages/` (TypeScript — done in s0.1)
- `web/` or `mobile/`
- Any service-specific code (`backend/services/`)
- Keycloak realm configuration (Phase 1)
