# Session: Monorepo Structure & Shared TypeScript Packages
## Phase
0 — Foundation

## Context files to load
- `SPEC.md` (sections: Monorepo Structure, Tech Stack, Shared packages)
- `CLAUDE.md`

## Your task
Scaffold the complete pnpm monorepo structure with root `package.json` (workspaces), and create all four shared TypeScript packages (`@keel/types`, `@keel/api`, `@keel/validation`, `@keel/utils`) with their `package.json`, `tsconfig.json`, and empty barrel `src/index.ts`. In `@keel/types`, define the initial TypeScript interfaces for `Company`, `UserProfile`, `Address`, and the `InvoiceStatus` union type exactly as specified in SPEC.md. In `@keel/utils`, add currency formatting (GBP pence-to-pounds), ISO 8601 date helpers, and a `formatInvoiceNumber(year, seq)` function.

## Constraints
- `@keel/api`, `@keel/types`, `@keel/validation`, `@keel/utils` must be importable by both `web` and `mobile`
- All packages use TypeScript strict mode
- No runtime dependencies in `@keel/types` (types only)
- `@keel/utils` must use integer pence arithmetic — no float money math (FP-2)
- pnpm workspaces: `packages/*`, `web`, `mobile`

## Done when
- `pnpm install` at repo root completes with zero errors
- `pnpm --filter @keel/types build` succeeds
- `pnpm --filter @keel/utils build` succeeds
- `pnpm --filter @keel/validation build` succeeds
- `pnpm --filter @keel/api build` succeeds
- `make verify` runs `pnpm -r build` and exits 0

## Do NOT touch
- `web/` directory (not yet created)
- `mobile/` directory (not yet created)
- Backend services (Phase 1+)
- Docker Compose (s0.2)
- Any database files
