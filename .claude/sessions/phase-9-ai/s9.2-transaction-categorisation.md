## Your task
Implement transaction auto-categorisation at `POST /api/v1/transactions/{id}/categorise`: (1) Remove the 501 stub. (2) `backend/services/banking/categoriser.py` — `TransactionCategoriser` class with `async categorise(transaction: Transaction) -> CategorisationResult`. Builds a prompt from `transaction.merchant_name` and `transaction.description`, calls Claude API requesting structured JSON output: `{ category: string, confidence: float, reasoning: string }`. Maps the response to a valid HMRC category from `CT_ALLOWABLE_CATEGORIES` (plus `ENTERTAINMENT`, `OTHER`). (3) The endpoint logic: confidence ≥ 0.85 → auto-apply the category and write audit log; confidence < 0.85 → return suggestion without applying, set `transaction.needs_review = True`. (4) Offload the Claude API call to a Celery task (`categorise_transaction_task`) — return `{ task_id, status: "processing" }` immediately (FP-12). Client polls `GET /tasks/{task_id}` for result. (5) Publish `transaction.categorised` event on completion.

## Constraints
- Claude API call must be in a Celery task — never blocking the request handler (FP-12)
- Response category must be validated against the HMRC category list — Claude may hallucinate invalid categories
- `ENTERTAINMENT` in response must set `flagged_as_entertainment=True` on the transaction (FR-2)
- Hardcoded prompt text is acceptable; hardcoded category names are not — use constants (FP-9)
- Rate limit: 100 calls/day per company via `check_rate_limit()` from s9.1
- Audit log written when category is auto-applied (confidence ≥ 0.85) (FP-5)
- `transaction.categorised` event published via outbox (FP-8)

## Done when
- `POST /transactions/{id}/categorise` returns 501 when `FEATURE_AI_ENABLED=false`
- With flag on: returns `{ task_id, status: "processing" }` immediately
- Celery task calls Claude and auto-applies category when confidence ≥ 0.85
- Celery task returns suggestion (no auto-apply) when confidence < 0.85
- Invalid category from Claude is mapped to `OTHER` + `needs_review=True`
- `pytest backend/services/banking/tests/test_categoriser.py -v` passes (mock Claude API)
- `make verify` runs pytest and exits 0
