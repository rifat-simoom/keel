## Your task
Implement the tax chat assistant at `POST /api/v1/ai/chat`: (1) Create `backend/services/ai/chat.py` — `TaxChatService` with `async stream_chat(company_id, message, session_id) -> AsyncIterator[str]`. Uses Claude's function calling with these read-only tool functions scoped to `company_id`: `get_balance()`, `get_ct_estimate()`, `get_vat_liability()`, `get_expenses_by_category()`, `get_invoices_outstanding()`. Each tool function queries the DB **read-only** scoped to `company_id` — no writes, ever. (2) The endpoint streams the response using Server-Sent Events (SSE) via FastAPI `StreamingResponse`. Client receives chunks as `data: {chunk}\n\n`. (3) Create `backend/services/ai/chat_tools.py` — the five tool implementations as async functions, each accepting `company_id: UUID` and returning structured data (never raw SQL results — always formatted summaries). (4) Store `chat_history` in Redis keyed by `session_id` (TTL 1 hour) to support multi-turn conversation.

## Constraints
- Tool functions are **read-only** — no DB writes, no mutations, no event publishing (SPEC requirement)
- Tool functions always filter by `company_id` — cross-tenant data access is impossible (FP-3)
- Never include raw SQL query results in the Claude context — format data first
- Never log chat messages (contain financial data + PII) — log session_id and company_id only (FP-18)
- Rate limit: 20 chat messages/day per company
- SSE stream: `Content-Type: text/event-stream`, `Cache-Control: no-cache`
- On Claude API error: stream a graceful error message — never expose API error details to client
- System prompt must include: company's current financial context (balance, CT estimate, next deadline)

## Done when
- `POST /ai/chat` returns 501 when `FEATURE_AI_ENABLED=false`
- With flag on: streams SSE chunks in real-time
- Each tool function returns scoped data for the correct company
- Multi-turn: second message in same session has access to first message context
- Asking "What's my CT liability?" calls `get_ct_estimate()` and incorporates the result
- `pytest backend/services/ai/tests/test_chat.py -v` passes (mock Claude API + assert tool calls)
- `make verify` runs pytest and exits 0

## Do NOT touch
- Transaction categorisation (s9.2)
- Receipt extraction (s9.3)
- UI — "✦ Ask Keel AI" button in Phase 6 web/mobile UI activates automatically when flag is on; implement the chat UI components in s9.5
