# Session: AI Layer — Feature Flag & Shared AI Infrastructure
## Phase
9 — AI Layer (Post-MVP)

## Context files to load
- `SPEC.md` (section: Phase 9 — Feature Flag, 9.1, 9.2, 9.3)
- `CLAUDE.md`
- `backend/shared/constants/tax.py`

## Your task
Set up the shared AI infrastructure that all three Phase 9 features depend on: (1) `backend/shared/ai/client.py` — Anthropic client factory: reads `ANTHROPIC_API_KEY` from env, returns a configured `anthropic.AsyncAnthropic` client. Never instantiate inline — always use this factory. (2) `backend/shared/ai/feature_flag.py` — `ai_enabled() -> bool` reads `FEATURE_AI_ENABLED` env var (case-insensitive `true`/`1` → True, anything else → False). FastAPI dependency `require_ai_enabled` that raises `HTTP 501` with `{"error": "AI features not yet available", "code": "FEATURE_DISABLED"}` if flag is off. (3) `backend/shared/ai/rate_limiter.py` — Redis-backed rate limiter: `check_rate_limit(company_id, feature, max_calls, window_seconds)` that raises `HTTP 429` with `Retry-After` header if exceeded. Default limits: categorisation 100/day, extraction 50/day, chat 20/day. (4) Update the three stub endpoints from Phases 3, 5, 6 to use `Depends(require_ai_enabled)` — they already return 501 without it, now it's consistent.

## Constraints
- `ANTHROPIC_API_KEY` must never appear in logs, responses, or error messages (FP-18)
- `ai_enabled()` reads the env var fresh each call — no module-level caching (allows runtime toggle)
- Rate limits per `company_id` — not per user (one limit per organisation)
- Redis key pattern: `ai_rate:{company_id}:{feature}:{date}` — expires at midnight UTC
- The three stub endpoints (`/transactions/{id}/categorise`, `/documents/{id}/extract`, `/ai/chat`) must all use `Depends(require_ai_enabled)` — no duplicate flag-checking logic

## Done when
- `ai_enabled()` returns False when `FEATURE_AI_ENABLED` is unset or `false`
- `ai_enabled()` returns True when `FEATURE_AI_ENABLED=true`
- `Depends(require_ai_enabled)` returns 501 when flag is off
- Rate limiter blocks the 101st call for a company on categorisation (100/day limit)
- `pytest backend/shared/tests/test_ai_infra.py -v` passes all flag and rate limiter tests
- `make verify` runs pytest and exits 0

## Do NOT touch
- Transaction categorisation logic (s9.2)
- Receipt extraction logic (s9.3)
- Tax chat logic (s9.4)
- Any UI changes — buttons are already rendered from earlier phases
