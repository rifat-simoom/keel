## Your task
Implement receipt data extraction at `POST /api/v1/documents/{id}/extract`: (1) Remove the 501 stub. (2) `backend/services/documents/extractor.py` ‚Äî `ReceiptExtractor` class: (a) Fetch the document from MinIO, (b) if image: run Tesseract OCR via `pytesseract` to get raw text, (c) pass raw text to Claude API with a structured output prompt requesting: `{ amount: string, date: string, vendor: string, vat_amount: string, category: string, confidence: { amount: float, date: float, vendor: float, vat_amount: float } }`, (d) return an `ExtractionResult` with all fields and per-field confidence scores. (3) Endpoint: offload to Celery task, return `{ task_id, status: "processing" }` (FP-12). On task completion: `PATCH /documents/{id}` with extracted fields, set low-confidence fields (< 0.8) to `needs_review=True`, publish `document.parsed` event.

## Constraints
- Tesseract OCR: call via `pytesseract.image_to_string()` in the Celery task ‚Äî not in the request handler (FP-12)
- `amount` and `vat_amount` from Claude must be parsed to `Decimal` ‚Äî reject non-numeric (FP-2)
- `vat_amount` must not exceed `amount √ó 0.20` ‚Äî if it does, set both fields to needs_review (FR-13)
- Category from Claude validated against HMRC category list ‚Äî invalid ‚Üí `OTHER` + `needs_review=True`
- Rate limit: 50 extraction calls/day per company
- `document.parsed` event published via outbox (FP-8)
- Audit log written when extraction results are applied to the document (FP-5)

## Done when
- `POST /documents/{id}/extract` returns 501 when `FEATURE_AI_ENABLED=false`
- With flag on: returns `{ task_id, status: "processing" }` immediately
- Celery task runs OCR, calls Claude, patches the document with extracted fields
- Low-confidence fields set to `needs_review=True`
- `vat_amount > amount √ó 0.20` triggers both fields to `needs_review`
