## Your task
Create the Alembic migration and SQLAlchemy 2.0 async models for `invoices` and `invoice_events` tables exactly as in SPEC.md Phase 4. Add `deleted_at`, `created_by`, `updated_by` to `invoices`. Add a `UNIQUE` constraint on `(company_id, invoice_number)` to enforce no duplicate invoice numbers per company. Add indexes on `invoices.company_id`, `invoices.status`, `invoices.due_date`, `invoices.issue_date`. Also add `currency TEXT NOT NULL DEFAULT 'GBP'` to `invoices` (FR-24). Update `packages/types/src/invoice.ts` with the `Invoice` and `LineItem` interfaces from SPEC.md (change `currency: 'GBP'` to `currency: string` per FR-24 Gap 6 decision).

## Constraints
- `invoices.invoice_number` is UNIQUE per company — DB-level constraint (FR-10)
- `invoices.status` default: `'draft'`
- `invoices.subtotal`, `vat_amount`, `total` all `NUMERIC(12,2)` — never FLOAT (FP-2)
- `line_items` stored as JSONB — must validate structure in the Python model `__init__`
- `invoice_events` records all status transitions — `event_type` TEXT, `metadata` JSONB
- Migration must add FK `transactions.invoice_id REFERENCES invoices(id)` (already stubbed in Phase 3)
- Migration reversible
