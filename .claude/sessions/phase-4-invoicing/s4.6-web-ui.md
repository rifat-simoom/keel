# Session: Invoicing — Web UI
## Phase
4 — Invoicing

## Context files to load
- `SPEC.md` (section: Phase 4 — Invoicing)
- `CLAUDE.md`
- `packages/types/src/invoice.ts`

- `web/src/pages/InvoicesPage.tsx`

## Your task
Build the invoicing web UI: (1) `web/src/pages/InvoicesPage.tsx` — invoice list with status filter tabs (All, Draft, Sent, Overdue, Paid), search by client name, sortable columns (date, amount, status). Status badges colour-coded: draft=grey, sent=blue, viewed=purple, paid=green, overdue=red, cancelled=slate. (2) `web/src/pages/InvoiceCreatePage.tsx` — multi-section form: client details, line items (dynamic add/remove rows with auto-calculated subtotals), VAT toggle per line, notes, due date. React Hook Form + Zod validation. Auto-saves as draft. (3) `web/src/pages/InvoiceDetailPage.tsx` — read-only view for sent/paid invoices. Action buttons: "Send" (draft only), "Mark Paid" (sent/viewed/overdue), "Download PDF", "Cancel" (sent+ shows "Cancel — issue credit note"). "Delete" button only visible for draft status. (4) `web/src/components/invoices/InvoiceSendModal.tsx` — confirmation modal that shows preview of VAT fields before sending.

## Constraints
- Invoice number field is read-only in the form — system-assigned, never user-editable (FR-10, FP-11)
- "Cancel" on a sent invoice must label as "Cancel (issue credit note)" — not just "Cancel" (FR-9)
- "Delete" button only renders for `status === 'draft'` — never for other statuses (FR-9)
- Line item totals must use integer pence arithmetic in the component — no float math (FP-2)
- VAT amounts must recalculate reactively as user changes quantities and prices
- Form validates with Zod schema from `@keel/validation` before submission
- All API calls via TanStack Query mutations with loading/error states

## Done when
- Invoice list loads with filtered tabs and search
- Create form calculates VAT and total in real-time as line items are entered
- Sending an invoice from draft shows the confirmation modal
- "Cancel" button for sent invoice is labelled correctly with credit note mention
- Draft invoice has a red "Delete" button; sent invoice does not
- `pnpm --filter web test --run` passes invoice form and list tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Mobile invoice UI (s4.7)
- PDF rendering (served from backend)
- Email sending (handled by backend send endpoint)
