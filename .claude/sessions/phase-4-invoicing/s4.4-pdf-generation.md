## Phase
4 — Invoicing

## Your task
Implement invoice PDF generation in `backend/services/invoice/pdf.py`: Create `generate_invoice_pdf(invoice: Invoice, company: Company) -> bytes` using `weasyprint` or `reportlab`. The PDF template must include every field required by UK VAT law (FR-8): invoice number, supplier name/address/VAT number, invoice date and tax point date, client name and address, line items table (description, quantity, unit price, VAT rate), subtotals (ex-VAT, VAT amount, total inc-VAT), for zero/exempt lines: reason stated. Include the company logo if `company.logo_key` is set (fetch from MinIO). Include payment terms and bank details from `company.invoice_footer`. The PDF is returned as bytes from the `GET /api/v1/invoices/{id}/pdf` endpoint.

## Constraints
- PDF must contain ALL 9 legally required VAT invoice fields (FR-8) — missing any makes the invoice legally invalid
- Invoice number must be prominently displayed (largest heading)
- If `company.vat_number` is null, the PDF must NOT show a VAT breakdown (unregistered company)
- Line item VAT rates must be shown per line — not just a total
- For zero-rated lines: reason must be stated (e.g., "Zero-rated — exported goods")
- Amounts formatted in GBP with £ symbol, 2 decimal places
- Logo fetched from MinIO at `company-logos/{company.logo_key}` — graceful fallback if missing
- PDF rendered server-side — no client-side rendering

## Done when
- `generate_invoice_pdf(invoice, company)` returns valid PDF bytes
- PDF opened in a viewer contains all 9 VAT required fields
- PDF for a zero-VAT invoice shows "Zero-rated" reason
- PDF for a company with no VAT number shows no VAT breakdown
- `pytest backend/services/invoice/tests/test_pdf.py -v` passes (field presence tests using pypdf or pdfminer to parse the output PDF)
- `make verify` runs pytest and exits 0

## Do NOT touch
- API endpoint for `/pdf` (already calls this function from s4.3)
- State machine or other invoice logic
- Web or mobile UI
