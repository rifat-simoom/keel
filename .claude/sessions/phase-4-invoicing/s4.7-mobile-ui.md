# Session: Invoicing — Mobile UI
## Phase
4 — Invoicing

## Context files to load
- `SPEC.md` (section: Phase 4 — Mobile Invoice Features)
- `CLAUDE.md`
- `packages/types/src/invoice.ts`
- `mobile/app/(app)/invoices/index.tsx`

## Your task
Build the mobile invoicing UI: (1) `mobile/app/(app)/invoices/index.tsx` — invoice list with status badge chips, swipe-right-to-send (draft invoices), swipe-left-to-mark-paid (sent/viewed/overdue), pull-to-refresh. (2) `mobile/app/(app)/invoices/create.tsx` — mobile-optimised create form: client name with autocomplete from previous invoices, line items, VAT, due date. Scrollable single-page layout (no tabs). (3) `mobile/app/(app)/invoices/[id].tsx` — invoice detail screen: status badge, line items summary, action buttons (Send, Mark Paid, Download/Share PDF, Cancel). "Share PDF" uses the native iOS/Android share sheet via `expo-sharing`. (4) Push notification received for `invoice.viewed` event deep-links to this screen.

## Constraints
- Swipe gestures via `react-native-gesture-handler` — not custom PanResponder
- "Cancel" button label must include credit note reference — "Cancel (credit note)" (FR-9)
- Line item arithmetic uses integer pence — no float (FP-2)
- Client autocomplete suggests names from previous invoices (fetched via TanStack Query)
- PDF share opens native share sheet with the PDF binary from `GET /invoices/{id}/pdf`
- Invoice number is read-only — system assigned (FR-10)
- Status-based action buttons: draft shows Send; sent/overdue shows Mark Paid; all sent+ show Cancel

## Done when
- Invoice list loads with status badges and pull-to-refresh
- Swipe-to-send transitions draft invoice to sent (with confirmation)
- Create form calculates totals in real-time
- Share PDF opens native share sheet on device
- `pnpm --filter mobile test --run` passes invoice list and detail screen tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web invoice UI (s4.6)
- Backend endpoints (already complete)
- Push notification service (Phase 7)
