## Your task
Implement all 9 invoice API endpoints: `GET /api/v1/invoices` (paginated, filterable by `status`, `date_from`, `date_to`, `client_name`), `POST /api/v1/invoices` (create draft — system assigns invoice number), `GET /api/v1/invoices/{id}`, `PUT /api/v1/invoices/{id}` (draft only — rejects if status != draft), `POST /api/v1/invoices/{id}/send` (validates VAT requirements, transitions via state machine, publishes `invoice.sent` event, emails PDF), `POST /api/v1/invoices/{id}/mark-paid` (transitions via state machine, publishes `invoice.paid` event), `POST /api/v1/invoices/{id}/cancel` (creates credit note record, transitions via state machine), `GET /api/v1/invoices/{id}/pdf` (returns PDF binary — generation in s4.4), `GET /api/v1/invoices/stats` (totals: outstanding, overdue, paid this month, paid this year).

## Constraints
- `POST /invoices` must call `generate_invoice_number()` — user cannot supply `invoice_number` (FP-11)
- `PUT /invoices/{id}` rejects with 409 if status != `draft` — cannot edit sent invoices (FR-9)
- `POST /{id}/send` calls `validate_vat_invoice_requirements()` and rejects with 422 if invalid (FP-22)
- `DELETE` on any invoice is forbidden — soft delete only for `DRAFT` status (FR-21)
- All state transitions via `InvoiceStateMachine` — never direct status assignment (FP-19)
- All mutations write audit log entries (FP-5)
- `POST /mark-paid` requires `Idempotency-Key` (FP-14)
- `invoice.paid` event must include `invoice_id`, `company_id`, `amount`, `currency` in payload
- Scope all queries to `current_user.company_id` (FP-3)

## Done when
- `POST /invoices` returns invoice with system-assigned `INV-2025-001` number
- `PUT /invoices/{id}` on a SENT invoice returns 409
- `POST /{id}/send` without VAT number returns 422 with field errors
- `POST /{id}/mark-paid` is idempotent — bank balance only changes once
- `pytest backend/services/invoice/tests/test_router.py -v` passes all endpoint tests
- `make verify` runs pytest and exits 0

## Do NOT touch
- PDF binary generation internals (s4.4)
- Overdue scheduler (s4.5)
- Web or mobile UI
