## Your task
Implement the Celery beat scheduled task in `backend/services/invoice/tasks.py`: `mark_overdue_invoices()` — runs every hour via Celery beat, finds all invoices where `due_date < NOW()` and `status IN ('sent', 'viewed')`, transitions each to `OVERDUE` via `InvoiceStateMachine`, writes an `invoice_events` row for each transition, writes an audit log entry for each transition, and publishes `invoice.overdue` event via outbox for each invoice. Configure the Celery beat schedule in `backend/services/invoice/celery_config.py`.

## Constraints
- Task must use `InvoiceStateMachine.transition_to("overdue")` — never direct status assignment (FP-19)
- Audit log must be written for every status change (FP-5)
- Event published via outbox pattern — not direct RabbitMQ publish (FP-8)
- Task must be idempotent — running it twice in the same hour must not double-transition any invoice
- Use a DB transaction per invoice — one failure does not roll back other invoices
- Log the count of invoices transitioned at INFO level (invoice IDs only, never amounts — FP-18)
- Celery beat schedule: `crontab(minute=0)` — top of every hour

## Done when
- Celery beat schedule is registered and `celery inspect scheduled` shows the task
- Task transitions a SENT invoice with `due_date = yesterday` to OVERDUE
- Running the task twice does not change an already-OVERDUE invoice
- `invoice.overdue` event appears in RabbitMQ after the task runs
- `pytest backend/services/invoice/tests/test_scheduler.py -v` passes
- `make verify` runs pytest and exits 0

## Do NOT touch

- PDF generation
- Web or mobile UI
- Other Celery tasks (Phase 7+)
