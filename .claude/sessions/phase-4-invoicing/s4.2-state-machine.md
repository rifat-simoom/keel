## Your task
Implement two core domain components: (1) `backend/services/invoice/state_machine.py` — `InvoiceStateMachine` class that wraps an `Invoice` model instance and exposes `can_transition_to(new_status: str) -> bool` and `transition_to(new_status: str) -> None`. The transition to `SENT` must call `validate_vat_invoice_requirements()`. Transitioning to `PAID` must schedule a bank credit event. Illegal transitions must raise `InvalidStateTransitionError`. (2) `backend/services/invoice/number_generator.py` — async `generate_invoice_number(session, company_id, year) -> str` using `SELECT MAX(invoice_number) ... FOR UPDATE` with the `INV-YYYY-NNN` format. Gaps are impossible when this function is used correctly — enforce with a DB lock. Also create `generate_credit_note_number(session, company_id) -> str` for `CN-NNN`.

## Constraints
- State machine must enforce ALL valid transitions from FR-9 exactly — no other transitions allowed (FP-19)
- `validate_vat_invoice_requirements()` in `validators.py` must check all 9 VAT invoice fields from FR-8
- Invoice number generator uses `SELECT ... FOR UPDATE` to prevent concurrent race conditions (FR-10, FP-11)
- `DRAFT` is the only status from which an invoice can be deleted (soft delete)
- Transitioning from `SENT` or beyond to any editable state is forbidden
- `InvalidStateTransitionError` must include `from_status`, `to_status`, and `invoice_id` in its message

## Done when
- `InvoiceStateMachine(invoice).can_transition_to("paid")` returns True for SENT invoice
- `InvoiceStateMachine(invoice).can_transition_to("draft")` returns False for SENT invoice
- `InvoiceStateMachine(invoice).transition_to("invalid")` raises `InvalidStateTransitionError`
- `generate_invoice_number(session, company_id, 2025)` returns `INV-2025-001` for first invoice, `INV-2025-002` for second
- `validate_vat_invoice_requirements(invoice, company)` returns error list when VAT number missing
- `pytest backend/services/invoice/tests/test_state_machine.py -v` covers all valid + invalid transitions
- `make verify` runs pytest and exits 0

## Do NOT touch
- API endpoints (s4.3)
- PDF generation (s4.4)
- Scheduler (s4.5)
- Web or mobile UI
