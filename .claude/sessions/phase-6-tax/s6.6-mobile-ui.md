## Your task
Build the mobile tax UI under `mobile/app/(app)/more/tax.tsx` and sub-screens: (1) `tax.tsx` — tax home screen with three large tappable cards: VAT Returns, Corporation Tax, Salary Optimiser. CT liability shown on the card in real-time from `GET /ct/estimate`. (2) `tax/vat.tsx` — period selector (current + past quarters), box breakdown in a vertical list, "Submit" button with confirmation alert. Submitted periods show lock badge and reference. (3) `tax/ct.tsx` — CT estimate breakdown card, payment deadline countdown chip, CT600 deadline countdown chip. (4) `tax/optimiser.tsx` — slider for desired income (0–£200,000), results rendered as a comparison table with animated value changes. (5) AI hook on the tax home screen: `✦ Ask Keel AI` button — visible but disabled, tapping shows `Alert.alert("Coming soon", "AI features are being built")`. (6) Update `mobile/app/(app)/index.tsx` to wire `TaxCard` and `DeadlineCard` to live API data.

## Constraints
- "✦ Ask Keel AI" button must always render — show `Alert` on press, never hide it (SPEC AI stub)
- Slider for desired income: `@react-native-community/slider` or `react-native-reanimated`-based
- Submitted VAT returns: all input fields disabled + lock icon in the header (FR-21)
- Deadline chips must use the same urgency colour logic as web (FR-23)
- Pull-to-refresh on all tax screens
- All amounts via `@keel/utils` formatCurrency

## Done when
- Tax home screen shows CT liability live from API
- VAT submission flow works on mobile with confirmation alert
- Salary optimiser slider updates results in real-time
- "✦ Ask Keel AI" shows correct alert message when tapped
- `pnpm --filter mobile test --run` passes tax screen tests
- `make verify` runs jest and exits 0
