## Your task
Implement all tax API endpoints in `backend/services/tax/router.py`: (1) `GET /api/v1/tax/vat/current-period` ‚Äî calculate and return the VAT return for the current quarter (computed from `vat_stagger` + today's date), cached in Redis for 5 minutes. (2) `GET /api/v1/tax/vat/periods` ‚Äî list all past VAT return periods with `submitted_at` status. (3) `POST /api/v1/tax/vat/submit` ‚Äî locks the current period's return (sets `submitted_at = NOW()`), stores it in DB, publishes `tax.deadline_approaching` event stub. Returns 409 if already submitted (FP-6). Requires `owner` role. (4) `GET /api/v1/tax/ct/estimate` ‚Äî run CT estimator for current accounting year, cached 5 min. (5) `GET /api/v1/tax/pay/optimise?desired_income={amount}` ‚Äî run pay optimiser, no caching (fast computation). (6) `GET /api/v1/deadlines` ‚Äî all upcoming deadlines sorted by date. (7) `GET /api/v1/deadlines/next` ‚Äî single next deadline.

## Constraints
- `POST /vat/submit` returns 409 if `submitted_at` is already set ‚Äî immutable after submission (FR-21, FP-6)
- All queries scoped to `current_user.company_id` (FP-3)
- All endpoints require auth (FP-7)
- Redis cache key pattern: `tax:{company_id}:vat_current` and `tax:{company_id}:ct_estimate`
- Deadline calculation must use the company's `vat_stagger` and `year_end_month` from DB (FR-7, FR-23)
- `GET /deadlines` returns deadlines with `type`, `due_date`, `days_remaining`, `urgency` (`normal`/`warning`/`urgent`/`critical`)
- Urgency thresholds: >30d=normal, 7‚Äì30d=warning, 1‚Äì7d=urgent, <1d=critical (FR-23)
