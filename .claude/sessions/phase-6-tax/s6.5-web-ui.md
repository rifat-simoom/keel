6 — Tax & Accounting

## Your task
Build the complete tax web UI: (1) `web/src/pages/TaxPage.tsx` — three-panel layout with tabs: VAT Returns, Corporation Tax, Salary Optimiser. (2) VAT Returns tab: current quarter box breakdown (Box 1–7 with labels and amounts), period date range selector to view historical returns, "Submit VAT Return" button (confirm modal → calls `POST /vat/submit`), submitted returns are read-only with lock icon and submission reference. "✦ Ask Keel AI" button — disabled, tooltip "Coming soon". (3) Corporation Tax tab: running CT estimate card (gross income, total expenses, AIA deduction, taxable profit, CT rate chip, CT liability in large bold text), payment deadline countdown, CT600 deadline countdown. (4) Salary Optimiser tab: single input "Desired annual income £____", results table comparing optimal split vs all-salary, effective tax rate bar chart using shadcn/ui. (5) Wire `TaxCard` on dashboard to `GET /api/v1/tax/ct/estimate`. (6) Wire `DeadlineCard` on dashboard to `GET /api/v1/deadlines/next`.

## Constraints
- Submitted VAT returns must be read-only — no edit fields, lock icon visible (FR-21, FP-6)
- "Submit" confirmation modal must show full box breakdown before user confirms — no blind submission
- "✦ Ask Keel AI" renders as disabled with tooltip — never hidden (SPEC AI stub, CLAUDE.md rule 8)
- All monetary amounts formatted via `@keel/utils` — never raw numbers
- Salary optimiser results must show BOTH personal tax cost AND CT saving together — not just one
- Deadline urgency colours: >30d green, 7–30d amber, 1–7d orange, <1d red (FR-23)

## Done when
- VAT return page shows current quarter boxes with correct labels
- Submit button triggers modal, calls API, then shows locked view with reference
- CT estimate card on dashboard and tax page are both live from API
- Salary optimiser updates results in real-time as income input changes
- `pnpm --filter web test --run` passes VAT panel, CT card, and optimiser tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Mobile tax UI (s6.6)
- AI chat backend (Phase 9)
- Any backend calculations
