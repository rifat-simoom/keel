## Your task
Implement `backend/services/tax/pay_optimiser.py` — `PayOptimiser` class with method `optimise(desired_income: Decimal) -> PayRecommendation`. Given a desired take-home income, it must calculate the most tax-efficient split between salary and dividends for a UK limited company director (2024/25 rates). Return a `PayRecommendation` dataclass containing: `optimal_salary` (up to £12,570 personal allowance), `optimal_dividends`, `total_income_tax`, `total_employee_nic`, `total_employer_nic` (company cost), `total_ct_saving` (salary reduces CT), `net_take_home`, `effective_tax_rate`, and a `comparison` dict showing the cost of taking everything as salary vs the optimal split. The optimiser must model **both** the personal tax and the CT impact together — not just one side.

## Constraints
- All arithmetic: `Decimal` only (FP-2)
- All thresholds/rates from `shared.constants.tax` (FP-9): `PERSONAL_ALLOWANCE = Decimal("12570")`, `DIVIDEND_ALLOWANCE = Decimal("500")`, `BASIC_RATE_BAND = Decimal("37700")`, `DIVIDEND_BASIC_RATE = Decimal("0.0875")`, `DIVIDEND_HIGHER_RATE = Decimal("0.3375")`, `NIC_PRIMARY_THRESHOLD = Decimal("12570")`, `NIC_EMPLOYEE_RATE_LOWER = Decimal("0.08")`, `NIC_EMPLOYER_RATE = Decimal("0.138")`
- Optimal salary = £12,570 (uses personal allowance, zero income tax, zero NIC at primary threshold)
- Dividends above £37,700 enter higher rate band — must correctly apply two tiers
- `net_take_home` must equal `desired_income - total_income_tax - total_employee_nic` (verify algebraically)
- `total_employer_nic` is a **company cost** — not deducted from take-home, shown separately

## Done when
- `optimise(Decimal("50000"))` returns `optimal_salary = 12570`, `optimal_dividends = 37430` with correct NIC calculations
- `optimise(Decimal("12570"))` returns `optimal_salary = 12570`, `optimal_dividends = 0`, zero tax
- Effective tax rate for £50,000 total income is demonstrably lower than "all salary" scenario
- `comparison` dict shows salary-only scenario alongside optimal for the same desired income
- `pytest backend/services/tax/tests/test_pay_optimiser.py -v` covers basic rate, higher rate band spillover, and minimum income edge cases
- `make verify` runs pytest and exits 0
