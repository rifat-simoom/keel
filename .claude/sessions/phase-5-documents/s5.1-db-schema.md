# Session: Documents — Database Schema & Models
## Phase
5 — Documents & Receipts

## Context files to load
- `SPEC.md` (section: Phase 5 — Database)
- `CLAUDE.md`
- `.claude/guardrails/financial-rules.md` (FR-13, FR-21, FR-22)

## Your task
Create the Alembic migration and SQLAlchemy 2.0 async model for the `documents` table as specified in SPEC.md Phase 5. Add `deleted_at`, `created_by`, `updated_by`. Add `amount NUMERIC(12,2)`, `currency TEXT DEFAULT 'GBP'`, `vendor_name TEXT`, `document_date DATE`, `vat_amount NUMERIC(12,2)`, `category TEXT`, `needs_review BOOLEAN DEFAULT TRUE`, `receipt_required BOOLEAN GENERATED ALWAYS AS (amount > 25) STORED` (or a trigger if generated columns not supported) to capture the manual-entry fields. Also add the FK column `receipt_id UUID REFERENCES documents(id)` to the `transactions` table (the stub from Phase 3). Add indexes on `documents.company_id`, `documents.status`, `documents.transaction_id`, `documents.category`.

## Constraints
- `documents.status` values: `processing`, `uploaded`, `matched`, `attention_required` — use CHECK constraint
- `amount` and `vat_amount` are `NUMERIC(12,2)` — never float (FP-2)
- `deleted_at` present — receipts can never be hard-deleted (FR-21, FR-22)
- `currency` column required (FR-24, FP-20)
- `receipt_required` flag: if `amount > 25` then True — implements FR-13 business rule at DB level
- Migration must add `transactions.receipt_id` FK (backwards-compatible — nullable)
- Migration reversible

## Done when
- `alembic upgrade head` adds documents table and transactions.receipt_id FK cleanly
- `alembic downgrade -1` reverses cleanly
- `pytest backend/services/documents/tests/test_models.py -v` passes (create document, FK to transactions, receipt_required flag, soft delete, currency present)
- `make verify` runs alembic + pytest and exits 0

## Do NOT touch
- Upload API (s5.2)
- Manual entry API (s5.3)
- Transaction matching (s5.4)
- Web or mobile UI
