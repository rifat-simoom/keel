# Session: Documents — Web UI
## Phase
5 — Documents & Receipts

## Context files to load
- `SPEC.md` (section: Phase 5, Gap 5 — Bank Reconciliation UI)
- `CLAUDE.md`
- `packages/types/src/index.ts`
- `web/src/pages/DocumentsPage.tsx`
- `web/src/pages/TransactionsPage.tsx`

## Your task
Build the documents web UI: (1) `web/src/pages/DocumentsPage.tsx` — drag-and-drop file upload zone (using `react-dropzone`), document list with status badges (`uploaded`, `matched`, `attention_required`), "needs review" red badge on unreviewed documents. (2) `web/src/components/documents/DocumentDetail.tsx` — slide-out sheet showing the document image/PDF preview (via pre-signed URL), manual entry form (amount, vendor, date, VAT amount, category dropdown), and the AI hook: `✦ Extract with AI` button — disabled, tooltip "Coming soon". (3) `web/src/components/documents/MatchTransactionModal.tsx` — shows a list of unmatched transactions filtered by proximity to the document amount and date, allowing user to click one to match. (4) Update `web/src/pages/TransactionsPage.tsx` — add a "Attach receipt" button on each transaction row that opens the document upload/match flow. (5) `web/src/components/dashboard/ReconciliationBadge.tsx` — red badge on dashboard showing unreconciled count from `GET /reconciliation/unmatched`.

## Constraints
- "✦ Extract with AI" must render as a disabled button with "Coming soon" tooltip — never hidden (SPEC AI stub)
- `ENTERTAINMENT` category in the dropdown must show a warning icon and tooltip: "Not CT-deductible"
- `OTHER` category must show a warning: "Needs review — will block VAT return submission"
- Document image preview must use the pre-signed URL from `GET /documents/{id}` — never the MinIO URL directly
- Drag-and-drop accepts only jpeg, png, heic, pdf — validate client-side before upload
- All monetary inputs validated as numbers with 2 decimal places

## Done when
- Drag-and-drop upload works and document appears in list with `uploaded` status
- Manual entry form saves and changes status to no-longer-needs-review
- Match transaction modal links document to transaction
- Dashboard reconciliation badge shows correct count
- `pnpm --filter web test --run` passes document list, detail, and match modal tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Mobile camera flow (s5.6)
- AI extraction backend (Phase 9)
- Reconciliation full-page view (Gap 5 — that is a separate session)
