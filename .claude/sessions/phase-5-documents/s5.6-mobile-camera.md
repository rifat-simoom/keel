# Session: Documents — Mobile Receipt Scanning (Camera)
## Phase
5 — Documents & Receipts

## Context files to load
- `SPEC.md` (section: Phase 5 — Mobile Receipt Scanning)
- `CLAUDE.md`
- `packages/api/src/client.ts`
- `mobile/app/(app)/scan.tsx`

## Your task
Build the complete mobile receipt scanning flow in the Scan tab: (1) `mobile/app/(app)/scan.tsx` — requests camera permission via `expo-camera`, renders a camera view with a capture button, perspective-crop overlay guide. On capture: runs `expo-image-manipulator` to resize to max 1500px width and convert to JPEG, uploads via `POST /api/v1/documents/upload` (multipart). (2) `mobile/app/(app)/documents/[id].tsx` — receipt detail screen that opens immediately after upload with: image preview (using the pre-signed URL), empty manual entry form (amount, vendor, date, VAT, category picker), and the AI hook: `✦ Extract with AI` button that shows a "Coming soon" toast on press. (3) One-tap "Save & Match" button that calls `PATCH /documents/{id}` then opens the transaction match picker. (4) `mobile/app/(app)/documents/index.tsx` — document list with status badges and pull-to-refresh.

## Constraints
- Camera permission request must use `expo-camera`'s permission hook — show a friendly message if denied
- `expo-image-manipulator` must compress to <1MB before upload (resize + JPEG quality 0.8)
- "✦ Extract with AI" shows a `Toast` / `Alert` with "Coming soon" message — never calls the AI endpoint (SPEC AI stub)
- Upload uses multipart form data — not base64 encoding
- Category picker: bottom sheet with HMRC category list and icons
- `ENTERTAINMENT` category shows a warning message inline: "Not CT-deductible"
- Handle upload errors with a retry button — don't silently fail

## Done when
- Tapping Scan tab opens the camera view
- Capturing a photo uploads and navigates to receipt detail screen
- Manual entry form saves correctly
- "✦ Extract with AI" shows "Coming soon" toast
- `pnpm --filter mobile test --run` passes scan flow and document detail tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web documents UI (s5.5)
- AI extraction backend (Phase 9)
- Transaction matching backend (already done in s5.4)
