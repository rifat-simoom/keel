## Your task
Implement the document upload API in `backend/services/documents/router.py`: `POST /api/v1/documents/upload` — accepts multipart form data (file + optional `transaction_id`), validates MIME type (accept: `image/jpeg`, `image/png`, `image/heic`, `application/pdf`), uploads to MinIO bucket `keel-documents` at key `{company_id}/{year}/{month}/{uuid}.{ext}`, creates a `documents` row with `status='uploaded'`, publishes `document.uploaded` event via outbox, returns the document record. Also implement `GET /api/v1/documents` (paginated list, filterable by `status`, `needs_review`, `date_from`), `GET /api/v1/documents/{id}` (includes a pre-signed MinIO URL valid for 1 hour), `DELETE /api/v1/documents/{id}` (soft delete only — sets `deleted_at`; MinIO file is NOT deleted).

## Constraints
- File size limit: 10MB — reject with 413 if exceeded
- MIME type whitelist: only jpeg, png, heic, pdf — reject others with 415
- MinIO key includes `company_id` prefix — cross-tenant file isolation (FP-3)
- `DELETE` is soft-only — MinIO file retained per FR-22 retention rules (FP-1)
- Pre-signed URL TTL: 3600 seconds — never expose MinIO credentials to client
- `document.uploaded` event published via outbox (FP-8)
- All endpoints require auth (FP-7)
- `POST /upload` requires `Idempotency-Key` (FP-14)

## Done when
- `POST /upload` with a JPEG uploads to MinIO and returns document record
- `POST /upload` with a 15MB file returns 413
- `POST /upload` with a .exe file returns 415
- `GET /documents/{id}` returns a pre-signed MinIO URL
- `DELETE /documents/{id}` soft-deletes the DB row but MinIO file still exists
- `pytest backend/services/documents/tests/test_upload.py -v` passes
- `make verify` runs pytest and exits 0
