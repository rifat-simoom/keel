## Your task
Implement two endpoints: (1) `PATCH /api/v1/documents/{id}` — updates the manual entry fields on an uploaded document: `amount`, `vendor_name`, `document_date`, `vat_amount`, `category`, `currency`. Sets `needs_review = False` when all required fields are present. Validates that `category` is one of the valid HMRC categories from `FR-2`. Writes audit log. (2) `POST /api/v1/documents/{id}/extract` — the AI extraction stub: checks `FEATURE_AI_ENABLED` env var; if false returns `501 Not Implemented` with body `{"error": "AI extraction not yet available", "code": "FEATURE_DISABLED"}`; if true (Phase 9) will call the extraction service. Also validate that `ENTERTAINMENT` category triggers a `flagged_as_entertainment: true` field in the response (never silent).

## Constraints
- `PATCH` must reject `category = "OTHER"` silently — instead set `needs_review = True`
- `ENTERTAINMENT` category: set a `flagged_as_entertainment: true` field — display a warning in UI (FR-2)
- `amount` must be stored as `Decimal` — reject float-like values that would lose precision (FP-2)
- `POST /extract` returns 501 when `FEATURE_AI_ENABLED` is not `true` (CLAUDE.md rule 8)
- Audit log written on every PATCH (FP-5)
- All fields validated with Pydantic model — no raw dict manipulation
- `vat_amount` must not exceed `amount * 0.20` — reject with 422 if it does

## Done when
- `PATCH /documents/{id}` with valid fields updates the record and sets `needs_review=False`
- `PATCH /documents/{id}` with `category="ENTERTAINMENT"` sets `flagged_as_entertainment=True`
- `PATCH /documents/{id}` with `vat_amount` > 20% of amount returns 422
- `POST /documents/{id}/extract` returns 501 when `FEATURE_AI_ENABLED=false`
- `pytest backend/services/documents/tests/test_manual_entry.py -v` passes
- `make verify` runs pytest and exits 0

## Do NOT touch
- Upload logic (s5.2)
- Transaction matching (s5.4)
- AI implementation (Phase 9)
- Web or mobile UI
