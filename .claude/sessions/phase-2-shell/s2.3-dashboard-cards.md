# Session: UI Shell — Dashboard Cards (Web + Mobile)
## Phase
2 — UI Shell & Navigation

## Context files to load
- `SPEC.md` (section: Phase 2 — Dashboard Cards)
- `CLAUDE.md`
- `packages/types/src/index.ts`
- `web/src/pages/DashboardPage.tsx`
- `mobile/app/(app)/index.tsx`

## Your task
Build the dashboard card components for both web and mobile with stubbed/hardcoded data (real API wiring happens in later phases): (1) `web/src/components/dashboard/BalanceCard.tsx` — account balance, sort code, account number. (2) `web/src/components/dashboard/InvoicesCard.tsx` — outstanding invoices total £, overdue count with red badge. (3) `web/src/components/dashboard/TaxCard.tsx` — estimated CT liability running total. (4) `web/src/components/dashboard/DeadlineCard.tsx` — next deadline with countdown chip (days remaining). (5) Wire all four into `web/src/pages/DashboardPage.tsx` in a responsive 2-column grid. (6) Create the same four cards as React Native components in `mobile/components/dashboard/` and wire into `mobile/app/(app)/index.tsx`. All cards display static placeholder values — no API calls yet.

## Constraints
- Web cards use shadcn/ui `Card` component
- Mobile cards use React Native `View` + `Text` — not shadcn
- CT tax card must display the value with a "Estimated" label — never claim it's a final figure
- Overdue invoice badge must be red (`destructive` variant on web, red background on mobile)
- Deadline chip must show days remaining as an integer, with colour coding (green >30d, amber 7–30d, red <7d)
- All monetary values formatted with `@keel/utils` currency formatter — never raw number display
- Cards accept props — no internal hardcoded strings (testable)

## Done when
- Web dashboard shows 4 cards in a 2-column grid with placeholder values
- Mobile dashboard shows 4 stacked cards with placeholder values
- Deadline chip shows correct colour for each threshold when passed different day counts
- `pnpm --filter web test --run` passes card render + deadline colour tests
- `pnpm --filter mobile test --run` passes mobile card render tests
- `make verify` runs both vitest and jest and exits 0

## Do NOT touch
- Real API connections (Phase 3+)
- Sidebar or tab navigation
- Auth logic
