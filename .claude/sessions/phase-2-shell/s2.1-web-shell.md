## Your task
Build the web app shell: (1) Create `web/src/layouts/AppLayout.tsx` — fixed 240px left sidebar + main content area. Sidebar contains: Keel logo, nav links (Dashboard, Invoices, Transactions, Tax, Documents, Payroll, Calendar), bottom section with Settings link and user avatar + name from Zustand auth store. (2) Create `web/src/components/Sidebar.tsx` — nav item component with active state using React Router `NavLink`. (3) Wrap all authenticated routes in `AppLayout` inside `web/src/App.tsx`. (4) Create placeholder page components (`DashboardPage`, `InvoicesPage`, `TransactionsPage`, `TaxPage`, `DocumentsPage`, `PayrollPage`, `CalendarPage`, `SettingsPage`) — each renders just a heading for now. (5) Use shadcn/ui components and Tailwind CSS throughout. Sidebar must be responsive (collapses to icon-only on medium screens).

## Constraints
- Use shadcn/ui `Sheet` for mobile sidebar drawer (below md breakpoint)
- Active nav link must have distinct visual styling (not just underline)
- All pages wrapped in `ProtectedRoute` — unauthenticated users redirected to `/login`
- No data fetching in this session — placeholder text only
- Sidebar width exactly 240px on desktop (matches SPEC)
- Use `@keel/types` for any type imports — never define types locally

## Done when
- `http://localhost:5173` shows the sidebar layout when logged in
- All 8 nav links navigate to their respective placeholder pages without errors
- Sidebar collapses on mobile (below 768px) and shows a hamburger icon
- Active link is visually highlighted
- `pnpm --filter web test --run` passes sidebar render tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Auth logic (Phase 1)
- Dashboard cards (s2.3)
- Mobile navigation (s2.2)
- Any API calls or data fetching
