# Session: UI Shell — Mobile Bottom Tab Navigation
## Phase
2 — UI Shell & Navigation

## Context files to load
- `SPEC.md` (section: Phase 2 — Mobile Navigation)
- `CLAUDE.md`
- `mobile/app/_layout.tsx`
- `mobile/stores/authStore.ts`

## Your task
Build the mobile app shell using Expo Router file-based navigation: (1) Create `mobile/app/(app)/_layout.tsx` — bottom tab navigator with 5 tabs: Dashboard (home icon), Invoices, Scan (centre — camera icon, raised FAB style), Transactions, More. The Scan tab must be visually distinct (larger icon, background colour, slightly raised). (2) Create placeholder screens: `mobile/app/(app)/index.tsx` (Dashboard), `mobile/app/(app)/invoices/index.tsx`, `mobile/app/(app)/scan.tsx`, `mobile/app/(app)/transactions/index.tsx`, `mobile/app/(app)/more/index.tsx`. (3) The "More" screen is a vertical list linking to Tax, Documents, Payroll, Settings, Help. (4) Update `mobile/app/_layout.tsx` (root layout) to redirect unauthenticated users to `/(auth)/login` and authenticated users to `/(app)`. All placeholder screens show only a title text.

## Constraints
- Use Expo Router `Tabs` from `expo-router` — not React Navigation directly
- Scan tab must be in the centre position (index 2 of 5)
- Bottom tabs must be in the "thumb zone" — no hamburger menus
- Tabs styled with React Native StyleSheet — no Tailwind (React Native doesn't support it)
- No NativeWind in this session unless already set up in the project
- All screens wrapped in auth guard from `mobile/app/_layout.tsx`

## Done when
- App shows bottom tabs with 5 items after login
- Scan tab is visually raised/distinctive in the centre
- Tapping each tab navigates to the correct screen
- Unauthenticated access redirects to login
- `pnpm --filter mobile test --run` passes navigation rendering tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web navigation (s2.1)
- Dashboard card content (s2.3)
- Auth logic (Phase 1)
- Any real data or API calls
