3 — Banking & Transactions

## Your task
Build the mobile banking UI: (1) Update `mobile/app/(app)/index.tsx` — wire the `BalanceCard` component to `GET /api/v1/accounts/me`. (2) Build `mobile/app/(app)/transactions/index.tsx` — FlatList with infinite scroll, category filter chips at the top (horizontal scroll), pull-to-refresh. (3) Create `mobile/components/transactions/TransactionItem.tsx` — credit amounts green, debit red, category badge, merchant, date. (4) Create `mobile/app/(app)/transactions/[id].tsx` — transaction detail screen (bottom sheet using `@gorhom/bottom-sheet` or `react-native-modal`) with category picker and the AI hook: `✦ Suggest category` chip — disabled, tapping it shows a Toast "Coming soon". (5) Create `mobile/app/(app)/card.tsx` — virtual card display with freeze/unfreeze toggle with haptic feedback (`expo-haptics`).

## Constraints
- "✦ Suggest category" chip must be rendered but disabled with a "Coming soon" toast on press (SPEC AI stub)
- Category picker: bottom sheet with scrollable list of HMRC categories
- `Haptics.impactAsync(ImpactFeedbackStyle.Medium)` on freeze toggle press
- Pull-to-refresh on transaction list using `refreshControl` prop on FlatList
- All amounts formatted via `@keel/utils` — never raw number (FP-2)
- Infinite scroll: load next page when within 5 items of end (`onEndReachedThreshold`)
- Use `expo-haptics` for freeze toggle — haptic must fire on state change

## Done when
- Dashboard balance card shows real API balance
- Transaction list scrolls with infinite scroll and pull-to-refresh works
- Tapping a transaction opens detail sheet with category picker
- Card freeze toggle calls endpoint and haptic fires
- `pnpm --filter mobile test --run` passes transaction list and card screen tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web UI (s3.3)
- Receipt scanning (Phase 5 — Scan tab stays as placeholder)
- AI categorisation backend (Phase 9)
