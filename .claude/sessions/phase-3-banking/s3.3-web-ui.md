# Session: Banking — Web UI
## Phase
3 — Banking & Transactions

## Context files to load
- `SPEC.md` (section: Phase 3 — Web Features)
- `CLAUDE.md`
- `packages/types/src/index.ts`
- `packages/api/src/client.ts`
- `web/src/pages/DashboardPage.tsx`

## Your task
Build the banking web UI: (1) Update `web/src/pages/DashboardPage.tsx` — wire `BalanceCard` to `GET /api/v1/accounts/me` using TanStack Query. (2) Create `web/src/pages/TransactionsPage.tsx` — account header (balance, sort code, masked account number with copy button), infinite-scroll transaction list with search input and filters (category dropdown, date range picker, amount range). (3) Create `web/src/components/transactions/TransactionRow.tsx` — amount coloured green for credits, red for debits, category badge, merchant name, date. (4) Create `web/src/components/transactions/TransactionDetail.tsx` — slide-out sheet (shadcn `Sheet`) with category selector (dropdown of all HMRC categories), receipt attachment slot (renders "Attach receipt — Phase 5" placeholder), and the AI hook: `✦ Auto-categorise` button disabled with tooltip "Coming soon". (5) Create `web/src/pages/CardPage.tsx` — card image panel with last_four, expiry, freeze toggle, spending limit display.

## Constraints
- "✦ Auto-categorise" button must be rendered but disabled — `disabled` attribute + "Coming soon" tooltip (SPEC AI stub rule)
- Category selector options must exactly match the HMRC categories in `.claude/guardrails/financial-rules.md` FR-2
- Account number displayed masked: `•••• 1234` with a copy-to-clipboard button that shows the full number
- All monetary amounts formatted via `@keel/utils` formatCurrency — never raw number rendering (FP-2)
- TanStack Query cache: transactions list stale after 30 seconds
- Loading states: use shadcn `Skeleton` components
- Error states: show a user-friendly message, never a stack trace (FP-21)

## Done when
- Dashboard balance card shows real account balance from API
- Transaction list loads with infinite scroll and filters work
- Category can be changed from the detail sheet and updates via API
- Card freeze toggle calls freeze/unfreeze endpoint and reflects new state
- `pnpm --filter web test --run` passes transaction list + card panel tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Receipt attachment (Phase 5)
- AI categorisation backend (Phase 9)
- Mobile UI (s3.4)
