## Your task
Write the complete backend integration test suite for Phase 3: (1) `backend/services/banking/tests/test_integration.py` — testcontainers + real Postgres + RabbitMQ: test account creation (seeded per company), test `GET /transactions` pagination + each filter (date, category, amount range), test cross-tenant isolation (company A cannot see company B's transactions — FP-3), test `POST /transactions` idempotency (same `Idempotency-Key` returns same result, does not double-credit), test freeze/unfreeze audit log is written, test `GET /stats` returns correct aggregations with Decimal precision, test `transaction.created` event is published to RabbitMQ after `POST /transactions`. (2) `backend/services/banking/tests/test_models.py` — unit tests: balance is Decimal not float, amount sign convention (positive=credit, negative=debit), currency column exists and defaults to GBP.

## Constraints
- testcontainers required — no mocked DB (CLAUDE.md rule 9)
- Cross-tenant test: create two companies, assert company B cannot retrieve company A's transactions (FP-3)
- Idempotency test: call `POST /transactions` twice with same key, assert balance only changes once
- RabbitMQ test: consume the `transaction.created` event and assert payload structure
- All monetary assertions use `Decimal` comparison — never float (FP-2)
- Soft delete test: deleted transaction does not appear in list queries (FP-16)
