# Session: Auth — Integration Tests
## Phase
1 — Auth & Identity

## Context files to load
- `CLAUDE.md`
- `backend/services/auth/router.py`
- `backend/services/auth/models.py`
- `backend/shared/middleware/auth.py`
- `web/src/lib/auth.ts`
- `web/src/stores/authStore.ts`

## Your task
Write the complete test suite for Phase 1: (1) `backend/services/auth/tests/test_integration.py` — pytest + testcontainers: spin up real Postgres, test `POST /register` creates company + profile, test idempotency (second call is a no-op), test `GET /me` with valid/invalid/expired token, test `require_role` blocks unauthorized roles. (2) `web/src/lib/__tests__/auth.test.ts` — Vitest: test `generateCodeVerifier` produces correct length, test `generateCodeChallenge` produces valid S256 hash, test `buildAuthUrl` includes required PKCE params. (3) `web/src/stores/__tests__/authStore.test.ts` — Vitest: test store initial state, test setting/clearing access token.

## Constraints
- Backend tests use testcontainers — real Postgres, no mocks for DB (CLAUDE.md rule 9)
- Use `pytest-asyncio` for all async test functions
- Each test is independent — no shared mutable state between tests
- Never use production Keycloak in tests — use a mock JWT signed with a test key
- Coverage must reach 80% on `auth.py`, `router.py`, and `middleware/auth.py`
- Web tests use Vitest + jsdom — no browser required

## Done when
- `pytest backend/services/auth/tests/ -v --cov=backend/services/auth --cov-fail-under=80` exits 0
- `pnpm --filter web test --run --coverage` exits 0 with ≥80% on auth files
- All 12+ test cases pass (register, idempotency, me-valid, me-invalid, me-expired, role-owner, role-blocked, PKCE params, code-verifier length, code-challenge hash, store-set, store-clear)
- `make verify` runs all the above and exits 0

## Do NOT touch
- Keycloak realm config
- Any Phase 2+ code
- Mobile tests (covered in s1.5 already)
