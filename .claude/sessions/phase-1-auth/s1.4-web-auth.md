# Session: Auth — Web Auth Flow (PKCE + Protected Routes)
## Phase
1 — Auth & Identity

## Context files to load
- `SPEC.md` (sections: Phase 1 — Web Auth Flow, Shared API Client)
- `CLAUDE.md`
- `packages/api/src/client.ts`
- `packages/types/src/index.ts`

## Your task
Implement the complete web authentication flow in the `web/` React app: (1) Create `web/src/stores/authStore.ts` — Zustand store holding `accessToken` (string | null), `user` (UserProfile | null), `isAuthenticated`. (2) Create `web/src/lib/auth.ts` — PKCE helpers: `generateCodeVerifier()`, `generateCodeChallenge(verifier)`, `buildAuthUrl()`, `exchangeCodeForTokens(code, verifier)`. (3) Create `web/src/pages/LoginPage.tsx` — button that initiates PKCE flow, stores verifier in sessionStorage (not localStorage). (4) Create `web/src/pages/CallbackPage.tsx` — handles `/callback?code=...`, exchanges code, calls `POST /api/v1/auth/register`, sets access token in Zustand store, stores refresh token in httpOnly cookie via API call. (5) Create `web/src/components/ProtectedRoute.tsx` — redirects to login if not authenticated. (6) Complete `packages/api/src/client.ts` with the request interceptor (attach Bearer token) and response interceptor (silent refresh on 401).

## Constraints
- Access token stored in Zustand (memory) ONLY — never localStorage (FP-10)
- Refresh token must be httpOnly cookie — set by backend, not by JavaScript
- Code verifier stored in `sessionStorage` only for the duration of the PKCE flow, cleared after exchange
- Silent token refresh: on 401, call refresh endpoint, retry original request once
- `buildAuthUrl()` must use `VITE_KEYCLOAK_URL`, `VITE_KEYCLOAK_REALM`, `VITE_KEYCLOAK_CLIENT_ID` env vars
- PKCE code challenge method: `S256`

## Done when
- Navigating to a protected route when logged out redirects to `/login`
- Clicking "Sign in" redirects to Keycloak (correct URL with PKCE params visible in browser)
- After successful Keycloak login, callback route redirects to `/` with user in Zustand store
- `pnpm --filter web test --run` passes auth store and PKCE helper unit tests
- `make verify` runs vitest and exits 0

## Do NOT touch
- Mobile auth (s1.5)
- Backend service (s1.3)
- Dashboard or other pages beyond login/callback/protected-route
