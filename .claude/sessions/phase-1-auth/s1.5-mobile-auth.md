# Session: Auth — Mobile Auth Flow (expo-auth-session + PKCE)
## Phase
1 — Auth & Identity

## Context files to load
- `SPEC.md` (sections: Phase 1 — Mobile Auth Flow, Mobile Deep Link Setup)
- `CLAUDE.md`
- `packages/api/src/client.ts`
- `packages/types/src/index.ts`

## Your task
Implement the complete mobile auth flow in the Expo React Native app: (1) Configure `app.json` with `scheme: "com.keelapp"` for deep link handling. (2) Create `mobile/app/(auth)/login.tsx` — "Sign in" button using `useAuthRequest` from `expo-auth-session` with PKCE, pointing at Keycloak. (3) Create `mobile/app/(auth)/callback.tsx` — Expo deep link handler that exchanges code + PKCE verifier for tokens. (4) Create `mobile/stores/authStore.ts` (or share from `@keel/api`) — Zustand store for access token in memory. (5) Create `mobile/lib/tokenStorage.ts` — save/load refresh token using `expo-secure-store` (never access token). (6) Create `mobile/app/_layout.tsx` — root layout with auth guard: on app start, reads refresh token from SecureStore, silently re-authenticates, redirects to login if refresh fails. (7) Wire up the Axios client from `@keel/api` to use the mobile token store.

## Constraints
- Auth must use system browser (`useAuthRequest` from `expo-auth-session`) — no WebView (SPEC requirement)
- Access token in Zustand memory only — never `SecureStore` for access token (FP-10)
- Refresh token in `expo-secure-store` key: `"keel_refresh_token"`
- Scheme in `app.json` must be exactly `"com.keelapp"`
- PKCE method: `S256`
- On app restart: attempt silent re-auth before showing login screen (no flash of login)
- Same Axios client from `@keel/api` — not a new one

## Done when
- Tapping "Sign in" opens Keycloak in the system browser (Safari/Chrome Custom Tab)
- After successful login, app navigates to `/(app)/index` (dashboard)
- On app restart with valid refresh token, user is auto-logged-in without seeing login screen
- On app restart with expired/missing refresh token, user sees login screen
- `pnpm --filter mobile test --run` passes auth store and token storage unit tests
- `make verify` runs jest and exits 0

## Do NOT touch
- Web auth (s1.4)
- Backend service (s1.3)
- Any screen beyond login/callback/_layout
