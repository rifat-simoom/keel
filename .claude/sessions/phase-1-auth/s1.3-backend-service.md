# Session: Auth — Backend Auth Service & Middleware
## Phase
1 — Auth & Identity

## Context files to load
- `SPEC.md` (sections: Phase 1 — Backend Auth Service, Auth Middleware, Web Auth Flow steps 9–10)
- `CLAUDE.md`

- `backend/shared/models/audit.py`

## Your task
Implement the complete backend auth service: (1) Complete `backend/shared/middleware/auth.py` — `require_auth` FastAPI dependency that decodes the Bearer JWT, validates against Keycloak JWKS (fetched and cached in Redis for 24h), and returns a `CurrentUser` dataclass with `id`, `company_id`, `keycloak_id`, `role`. Implement `require_role(role: str)`. (2) Create the four auth endpoints in `backend/services/auth/router.py`: `POST /api/v1/auth/register` (upsert company + user_profile on first Keycloak login), `GET /api/v1/auth/me`, `PUT /api/v1/auth/me`, `POST /api/v1/auth/logout`. (3) Create `backend/services/auth/main.py` FastAPI app that mounts the router.

## Constraints
- JWKS cache in Redis: key `keycloak:jwks`, TTL 86400 seconds, refreshed on 401 from Keycloak
- `POST /register` must be idempotent — re-registering the same Keycloak ID is a no-op (upsert)
- `company_id` in `CurrentUser` extracted from JWT claims, never from request body (FP-4)
- All four endpoints require auth except `POST /register` (bootstrap call)
- `PUT /me` writes audit log entry (FP-5)
- Error shape: `{ "error": string, "code": string }` — never expose stack traces (FP-21)
- Use `python-jose` or `PyJWT` for JWT decode

## Done when
- `POST /api/v1/auth/register` with valid Keycloak JWT creates company + user_profile rows
- `GET /api/v1/auth/me` with valid token returns `{ id, email, full_name, role, company_id }`
- `GET /api/v1/auth/me` with expired token returns 401
- `GET /api/v1/auth/me` with no token returns 401
- `pytest backend/services/auth/tests/test_router.py -v` passes all cases
- `make verify` runs pytest and exits 0

## Do NOT touch
- Keycloak realm export (s1.2)
- Web auth flow (s1.4)
- Mobile auth flow (s1.5)
- Any other service
